# Contributing to Unified Bottle Detection System

Thank you for your interest in contributing to the Unified Bottle Detection System! We appreciate all contributions, whether they are bug reports, documentation improvements, or code enhancements.

## How to Contribute

### 1. Report Bugs
- Check if the bug has already been reported in Issues
- Provide a clear description of the bug
- Include steps to reproduce the issue
- Specify your environment (OS, Python version, dependencies)

### 2. Suggest Enhancements
- Use GitHub Issues to suggest new features
- Clearly describe the enhancement and its benefits
- Explain why this enhancement would be useful

### 3. Submit Pull Requests
- Fork the repository
- Create a new branch: `git checkout -b feature/your-feature-name`
- Make your changes
- Test your code thoroughly
- Commit with clear messages: `git commit -m "Add feature: description"`
- Push to your fork: `git push origin feature/your-feature-name`
- Open a Pull Request with a clear description

## Development Setup

```bash
# Clone the repository
git clone https://github.com/HATIMABDESSAMAD/detection--bottle--RECYBOT.git

# Create virtual environment
python -m venv venv

# Activate virtual environment
# On Windows:
venv\Scripts\activate
# On macOS/Linux:
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

## Code Guidelines

- Write clear and descriptive comments
- Follow PEP 8 style guidelines
- Test your code before submitting
- Update documentation as needed
- Ensure all tests pass

## Testing

```bash
# Run tests
python -m pytest

# Run specific test file
python -m pytest tests/test_filename.py
```

Feel free to open an Issue or contact the maintainers.
